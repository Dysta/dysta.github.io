Une API REST statique gratuitement

Une API REST statique : définition, cas d’usage et mise en place Aujourd’hui, je vais vous présenter une façon simple et gratuite de concevoir une API REST statique, hébergée via un simple repository GitHub. Qu’est-ce qu’une API REST statique ? Je définis une API REST statique comme une API qui respecte trois principes fondamentaux : Serverless : elle ne dépend d’aucun cloud provider (AWS, GCP, Azure, etc..). Stateless : elle ne conserve aucun état entre les requêtes. Scalable : elle peut être mise à l’échelle facilement, sans contrainte d’infrastructure. En pratique, il s’agit d’une API qui expose des données sous forme de fichiers JSON statiques, accessibles via des requêtes HTTP, sans traitement côté serveur.
